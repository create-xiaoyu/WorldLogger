# WorldLogger

> A NeoForge mod that logs important Minecraft world events to MySQL, allowing administrators to view them via commands, an in-game GUI, and an AI database assistant.

## Project Overview

WorldLogger is a logging mod for Minecraft NeoForge servers. It records events such as player logins, logouts, deaths, item drops, experience changes, chat messages, commands, container interactions, block placement/breaking, explosions, entity deaths, and entity spawns into a MySQL database. It provides three ways to view the logs:

- Chat commands: `/worldlogger select` and `/worldlogger search`
- Graphical interface: `/worldlogger gui`
- AI assistant: `/worldlogger ai ...`

Its core goal is to turn "what happened" in the Minecraft world into queryable, replayable, and analyzable data. The server acts like an observer, listening to key events in the game, organising them into database records, and helping administrators understand these records through commands, a GUI, and AI.

## Quick Demo

Follow these steps to quickly get started with WorldLogger:

1. Start MySQL and create a database (e.g., `world_logger`).
2. Start the NeoForge dedicated server and configure the database connection (see "Database Configuration" below).
3. Join the server and perform some actions that trigger logging, for example:
    - Log in
    - Send a chat message
    - Execute a command
    - Open a chest and move items
    - Break or place blocks
    - Drop an item
    - Spawn or kill an entity
4. Use the commands to view logs:
    - `/worldlogger select PLAYER_LOGIN_INFO 1` – view login records in the chat
    - `/worldlogger select PLAYER_CONTAINER_INFO 1` – view container interaction records
    - `/worldlogger search` – search for records within 16 blocks of your current position
    - `/worldlogger gui` – open the graphical interface to browse all tables
5. If you have configured the AI assistant (see "AI Configuration" below):
    - `/worldlogger ai Summarise what I have done since my last login`

## Main Features

### Event Types Logged

WorldLogger records the following events and stores them in separate database tables (full table list below):

- Player logins and logouts (with time, coordinates, IP)
- Player deaths (with damage source, death message)
- Player item losses (dropped items or death drops)
- Player experience changes (with amount and source)
- Chat messages (raw text and rich text components)
- Command executions (source: player, command block, console, etc.)
- Container interactions (item take/put/modify during opening)
- Block placement (by any entity)
- Block breaking (by players, non-player entities, or explosions)
- Non-player entity deaths and spawns

### Viewing Methods

#### 1. Chat Commands (Admin)

- `/worldlogger select <table> [page]` – view one record from a table per page, with clickable pagination.
- `/worldlogger search [page]` – search for all events within 16 blocks of your current position, sorted by time descending (available to regular players as well).

#### 2. Graphical Interface (Admin)

- `/worldlogger gui` – opens a visual browsing interface:
    - Left panel lists all queryable tables.
    - Top search bar allows filtering by player UUID, player name, or coordinates.
    - Right panel shows the full field content of the selected record, with long text wrapping and scrolling.

#### 3. AI Assistant (Admin)

- `/worldlogger ai <message>` – chat with the AI assistant. The server-side AI can call read‑only database tools to help you analyse logs.
- `/worldlogger ai reset` – clear the current player's AI conversation context and pending approval requests.
- `/worldlogger ai approve <ID>` – when the AI needs to read more data than the automatic limit, it generates an approval ID; use this command to approve and continue.

The AI assistant supports the following read‑only database tools:

- List all queryable tables
- Describe a table's columns and total row count
- Query specific rows from a table
- Search for records near the executing player
- Get a timeline of a player's activities after their latest login

If the AI requests a data volume exceeding the automatic limit (default 20 rows), administrator approval is required before execution.

## Configuration

WorldLogger's configuration file is automatically generated by NeoForge, usually in the `config/` folder or the world's `serverconfig/` folder. You need to set the following:

### Database Configuration

| Setting             | Default                       | Description                            |
| ------------------- | ----------------------------- | -------------------------------------- |
| `database_url`      | `jdbc:mysql://localhost:3306` | MySQL JDBC URL (without database name) |
| `database_name`     | `world_logger`                | Database name                          |
| `database_username` | `root`                        | MySQL username                         |
| `database_password` | (empty)                       | MySQL password                         |
| `thread_number`     | `8`                           | Number of database read/write threads  |

> [!IMPORTANT]
> **Database setup example:**
>
> ```sql
> CREATE DATABASE world_logger CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
> CREATE USER 'worldlogger'@'localhost' IDENTIFIED BY 'your_password';
> GRANT ALL PRIVILEGES ON world_logger.* TO 'worldlogger'@'localhost';
> FLUSH PRIVILEGES;
> ```
>
> Then fill in the corresponding username and password in the configuration file.

### AI Configuration (Optional)

AI functionality is disabled by default. To enable it, set the following in the configuration:

| Setting                     | Default                     | Description                                               |
| --------------------------- | --------------------------- | --------------------------------------------------------- |
| `enabled`                   | `false`                     | Enable AI features                                        |
| `api_base_url`              | `https://api.openai.com/v1` | OpenAI-compatible API base URL                            |
| `api_key`                   | (empty)                     | API key                                                   |
| `model`                     | (empty)                     | Model name (e.g., `gpt-4o`)                               |
| `max_auto_search_depth`     | `20`                        | Max rows AI can auto‑fetch; requires approval if exceeded |
| `max_approved_search_depth` | `256`                       | Max rows allowed after admin approval                     |
| `max_tool_iterations`       | `10`                        | Max AI tool call iterations                               |
| `max_output_tokens`         | `4096`                      | Max tokens in AI response                                 |
| `request_timeout_seconds`   | `60`                        | AI request timeout in seconds                             |
| `debug_log_payloads`        | `false`                     | Log AI request/response payloads (for debugging)          |

> Note: The AI API must be compatible with the OpenAI Responses API format. Keep your API key secure; it is not logged by default.

## Database Tables

WorldLogger automatically creates the following 15 tables on startup (if they do not exist):

| Table Name              | Content                                                               |
| ----------------------- | --------------------------------------------------------------------- |
| `PLAYER_BASE_INFO`      | Basic player info (UUID, name, total play time)                       |
| `PLAYER_LOGIN_INFO`     | Login time, IP, coordinates, world                                    |
| `PLAYER_LOGOUT_INFO`    | Logout time, coordinates, world                                       |
| `PLAYER_DEATH_INFO`     | Death type, coordinates, world, damage source, weapon, death message  |
| `PLAYER_LOST_ITEM`      | Dropped item or death drop item info (can link to death ID)           |
| `PLAYER_XP_INFO`        | Experience change type, source, amount, total XP, coordinates         |
| `EXECUTE_COMMAND_INFO`  | Command source, coordinates, world, raw command                       |
| `SERVER_CHAT_INFO`      | Player, coordinates, world, raw message and rich text component       |
| `PLAYER_CONTAINER_INFO` | Container ID, slot, before/after items, change type (take/put/modify) |
| `ENTITY_PLACE_BLOCK`    | Placing entity, coordinates, world, block ID and NBT                  |
| `PLAYER_BREAK_INFO`     | Player breaking block coordinates, world, block ID and NBT            |
| `ENTITY_BREAK_INFO`     | Non-player entity breaking block coordinates, world, block ID and NBT |
| `EXPLOSION_BREAK_BLOCK` | List of affected blocks (JSON), explosion source, position            |
| `ENTITY_DEATH_INFO`     | Non-player entity death info, damage source                           |
| `ENTITY_SPAWN_INFO`     | Non-player entity spawn info                                          |

All tables include a timestamp column for chronological queries.

## Building and Installation (for developers / server admins)

### Installing the Mod

1. Download `worldlogger-<version>.jar` from the releases page.
2. Place it in the NeoForge server's `mods/` folder.
3. Start the server – the configuration file will be generated automatically.
4. Edit the configuration to set your database and AI settings as needed.
5. Restart the server for changes to take effect.

### Building from Source

```bash
./gradlew build
```

The built JAR will be placed in `build/libs/`.

### Running in Development

- Client: `./gradlew runClient`
- Dedicated Server: `./gradlew runServer`

## Frequently Asked Questions

**Q: Does the client need to install this mod?**  
A: The client can run without it, but installing it enables the GUI and AI chat features. The server must have the mod installed.

**Q: Will large amounts of data cause server lag?**  
A: All database reads/writes are executed in separate thread pools and do not block the game's main thread. You can adjust `thread_number` according to your server's performance.

**Q: Can the AI assistant modify the database?**  
A: No. All AI tools are read‑only and will never execute any write or delete operations.

**Q: How do I reset the AI conversation?**  
A: Use `/worldlogger ai reset` to clear the current player's dialogue context.

**Q: Can the search radius be changed?**  
A: Currently fixed at 16 blocks; customisation may be added in the future if needed.
